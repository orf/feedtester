Feed creator

This is a very simple service that generates persistent updating RSS and ATOM feeds.

Usage:

A feed can be accessed by using any string as a key by these formats: {% url 'feeds:key' 'rss' 'KEY' %} or {% url 'feeds:key' 'atom' 'KEY' %}. For example: {% url 'feeds:key' 'rss' uuid.hex %} or {% url 'feeds:key' 'atom' uuid.hex %}

If you don't want to specify a key you can request {% url 'feeds:cookie' 'rss' %}. This will set a cookie which you will need to keep and re-send with each request.

Options

Message rate

By default a new item will be created when the feed is requested, if one hasn't been created in the last 30 seconds. You can pass an integer between 15 and 60 in the time parameter. To create a feed with a new entry every 20 seconds request this URL: {% url 'feeds:key' 'rss' uuid.hex %}?time=20

Simulate issues

Feeling brave? Append ?tryme (e.g {% url 'feeds:key' 'rss' uuid.hex %}?tryme) to any URL. This will sometimes cause one of the following things to happen:

  • Invalid XML
  • Internal server error
  • Non XML response
  • Non HTTP response
  • Redirect to another page
  • Out of order or missing results
  • Results with future dates
